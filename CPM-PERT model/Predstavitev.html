Predstavitev problema

  • Podan imamo projekt, ki je sestavljen iz mnoice n opravil. Za vsako opravilo o imamo podano trajanje, pa tudi mnoico drugih opravil, ki jih moramo opraviti pred zaetkom opravila o. Izraunati moramo as trajanja celotnega projekta.

  • Projekt si lahko predstavljamo kot usmerjen graf, kjer so opravila vozlia, povezava od vozlia i do j pa pomeni, da je opravilo i pogoj za opravilo j. Dodamo e zaetno in konno vozlie. Iz zaetnega vozlia gredo povezave v vsa vozlia, ki nimajo drugih vstopnih povezav, v konega pa gredo povezave iz vseh vozli, ki nimajo drugih izstopnih povezav.

Vhodni podatki

  • Vektor opravil: vektor tevil od 1 do n, e imamo n opravil
  • Seznam predhodnikov: seznam vektorjev, kjer vektor na poloaju i predstavlja mnoico opravil, ki so pogoj za opravilo i. e opravilo i nima pogojev, potem potem je na i-tem mestu vektor, ki vsebuje 0.
  • Vektor asov: vektor pozitivnih tevil, kjer tevilo na i-tem mestu predstavlja trajanje opravila i.

Matrika sosednosti

Graf lahko zapiemo kot matriko sosednosti. To je matrika sestavljena iz 0 in 1, kjer 1-ka na mestu [i,j] pove, da imamo povezavo, ki gre iz vozlia i v j.

V prvem delu najinega programa sestaviva matriko sosednosti:

Adj<-function(Opr,Pred)
  z<-length(Opr)+1
  NO<-c(0, Opr,z)
  A<-matrika velikosti (z+1)*(z+1)
  O<-Opr
  for i in Opr:
    p<-Pred[i]
    for j in p:
      A[j+1,i+1]<-1 
      O[!O %in% p]

Topoloko urejanje

  • Ko imamo graf predstavljen v obliki matrike, moramo doloiti vrstni red opravljanja opravil. Pri tem si pomagamo z algoritmom za topoloko urejanje.
top.sort<-function(Opr, Pred)
  adj<-Adj(Opr,Pred)
  z<-length(Opr)+1
  NO<-c(1, Opr+1,z+1)
  for v in NO:
    st[v]<-sum(adj[,v])
  for v in NO:
    if st[v]==0:
      Q<-c(Q,v)

Topoloko urejanje

  • Funkcija za topoloko urejanje nam vrne vektor zaporednih tevilk opravljanja (e imamo na i-tem mestu tevilko j, to pomeni da se bo i opravljal j-ti po vrsti). Bolje je, e imamo vektor, kjer si opravila sledijo v istem zaporedju kot se bodo opravljala. Zato uporabimo naslednjo funkcijo:
vrstni.red<-function(sez)
  j<-0
  zaporedje<-c()
  while j<length(sez)
    for i in 1:length(sez)
      if(sez[i]==j)
        zaporedje[j]<-i-1
        j<-j+1
  return(zaporedje)

Trajanje

  • Ko enkrat imamo topoloko urejanje, moramo le e izraunati as trajanja projekta. To naredimo s funkcijo trajanje:
trajanje<-function(Opr,Pred,Cas)
  vr<-vrstni.red(top.sort(Opr,Pred))
  cas[vr[1]]<-0 
  for i in 2:(length(vr)-1) 
    c.pred <- c(0, sapply(Pred[[vr[i]]], function(j) cas[j]+Cas[j]))
    cas[vr[i]]<-max(c.pred)
  
  a<-Adj(Opr,Pred)[,length(vr)+1]
  for i in 1:length(a):
    if a[i]==1:
      predh<-c(predh,(i-1))
  for j in predh:
    c1<-c(c1,cas[j]+Cas[j])
  konec<-max(c1)
  return(konec)

Premaknjena normalna porazdelitev

simul1 <- function(cas, sd, p) {
        sluc <- vector(length = length(cas))
        i <- 1
        for(c in cas){
                sluc[i] <- rtruncnorm(1, c * p, mean = c, sd = sd)
                i <- i+1
        }
        return(sluc)
}

simul2 <- function(cas, m) {
        if(m>2 | m < 0) {return(FALSE)}
        sluc <- vector(length = length(cas))
        n <- max(m, 2-m)
        m <- min(m, 2-m)
        i <- 1
        for(c in cas){
                sluc[i] <- runif(1, min = m*c, max = n * c)
                i <- i+1
        }
        return(sluc)
}

Binomska porazdelitev

simul3 <- function(cas, n) {
        sluc <- vector(length = length(cas))
        i <- 1
        for(c in cas){
                p <- c / n
                sluc[i] <- max(rbinom(1, n, p), round(0.8*c))
                i <- i+1
        }
        return(sluc)
}
